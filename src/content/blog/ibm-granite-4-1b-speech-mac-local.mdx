IBM just dropped Granite 4.0 1B Speech — a 1 billion parameter speech-to-text model that beats models twice its size on accuracy benchmarks. And it runs natively on Apple Silicon.

## Why This Matters

The speech recognition space has been dominated by two options:
1. **Cloud APIs** (OpenAI Whisper API, Google Speech-to-Text) — fast but privacy-invasive and costly at scale
2. **Local Whisper** — accurate but resource-hungry, especially the large-v3 model

Granite 4.0 1B Speech changes the equation. At just 1B parameters, it:

- **Ranks #1** on the HuggingFace Open ASR Leaderboard
- **Cuts size in half** compared to granite-speech-3.3-2b while improving accuracy
- **Supports 6 languages**: English, French, German, Spanish, Portuguese, Japanese
- **Runs on edge devices** — including M-series Macs via MLX

The Word Error Rate (WER) across benchmarks like LibriSpeech, GigaSpeech, and AMI is lower than models with 2-8x more parameters.

## Run It Locally on Your Mac

If you've got an M1/M2/M3/M4 Mac, you can run this in minutes.

### Step 1: Install mlx-audio

```bash
pip install -U mlx-audio
```

### Step 2: Transcribe Audio (CLI)

```bash
python -m mlx_audio.stt.generate \
  --model ibm-granite/granite-4.0-1b-speech \
  --audio "your-audio.wav" \
  --output-path "transcript" \
  --verbose
```

That's it. No API keys. No cloud. No subscription.

### Step 3: Python Integration

```python
from mlx_audio.stt.utils import load_model
from mlx_audio.stt.generate import generate_transcription

model = load_model("mlx-community/granite-4.0-1b-speech-8bit")
transcription = generate_transcription(
    model=model,
    audio="audio.wav",
    output_path="transcript.txt",
    format="txt",
    verbose=True,
)
print(transcription.text)
```

The 8-bit quantized version at `mlx-community/granite-4.0-1b-speech-8bit` cuts memory usage further — perfect for older Macs or background processing.

## Keyword Biasing: The Killer Feature

Here's something most speech models can't do: **keyword list biasing**.

You can pass a list of names, acronyms, or domain-specific terms, and Granite will prioritize recognizing them correctly. This is huge for:

- **Meeting transcription** (getting names right)
- **Technical content** (acronyms like "MLX", "vLLM", "ASR")
- **Medical/legal transcription** (specialized terminology)

Just append `Keywords: <kw1>, <kw2>` to your prompt.
